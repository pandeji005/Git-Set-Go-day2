Introduction to Linux Shell

Understanding the Linux Shell architecture, core types, and scripting definitions.

What is a Linux Shell?

A Shell is a specialized software utility that provides an interface/environment for users to execute commands and interact with the operating system's kernel. It sits between the user (typing commands or executing scripts) and the underlying operating system kernel, parsing command text and launching processes.

To visualize how the shell connects users to system hardware, examine the layered architecture below:

Linux System Layered Architecture

Different Types of Shells

Several command-line interpreters have evolved over the history of UNIX and Linux operating systems, offering different features and syntax:

  • sh (Bourne Shell): The original Unix shell. Extremely basic, but highly compatible; it serves as the baseline standard for POSIX compatibility.
  • bash (Bourne Again Shell): The default shell on most standard Linux distributions. It extends `sh` with custom syntax, autocomplete, and history tracking.
  • zsh (Z Shell): A highly customizable shell incorporating features from Bash, Ksh, and Tcsh. It is the default shell on modern macOS versions.
  • ksh (Korn Shell): Combines features of the classic Bourne shell and C shell, popular in commercial Unix environments.
  • tcsh (Tenex C Shell): Extends the classic C-shell (`csh`) using syntax similar to the C programming language.
  • fish (Friendly Interactive Shell): Focused on user-friendly interactive features like syntax highlighting out of the box, though it is not POSIX-compliant.

Checking Your Active Shell

You can query the shell type currently running in your command window by reading the positional parameter zero variable:

Terminal 
echo $0

This command will output the path of the active shell binary (e.g. `/bin/bash` or `/bin/zsh`).

What is Shell Scripting?

A Shell Script is a plain text file containing a sequence of operating system commands. When executed, the shell interpreter reads the file line-by-line and processes each command sequentially, as if you had typed them directly into the console.

Shell scripting is commonly used for:

  • System administration automation (backups, service management).
  • File system manipulations, text analysis, and log processing.
  • Application builds, software installations, and environment setup.
  • Interactive command-line menus for non-technical users.

Getting Started

Writing your first shell script, configuring execute permissions, and using comments.

Your First Basic Script

To write a shell script, create a plain text file (commonly ending in `.sh`, though not strictly required) and input the following lines:

hello_world.sh 
#!/bin/bash
echo "Hello World!"

What is the Shebang?

The shebang (#!/bin/bash) is a special character sequence at the very beginning of a script. It consists of the characters `#!` followed by the absolute path to the shell interpreter binary that should execute the script.

When you run a script directly (e.g. `./hello_world.sh`), the kernel reads the shebang line to identify which program to spawn (in this case, `/bin/bash`) to parse and execute the script content. Common shebang variants include:

  • #!/bin/bash: Explicitly uses the Bash shell.
  • #!/bin/sh: Uses the system's POSIX-compliant standard shell.
  • #!/usr/bin/env bash: Searches the user's active environment path to find and run `bash` (useful for cross-platform portability).

How to Execute a Script

By default, newly created text files do not have execute permissions. You must modify the file attributes before running the script directly:

1. Grant Execute Permissions

Use the `chmod` command to add the execute permission (`x`) for the current user:

Terminal 
chmod +x hello_world.sh

2. Choose an Execution Method

Once executable, you can run the script using any of the following directory paths:

Terminal 
# Method A: Execute relative path (requires ./ prefix if current dir is not in PATH)
./hello_world.sh

# Method B: Execute absolute system path
/home/paul/scripts/hello_world.sh

# Method C: Invoke the interpreter directly (does NOT require chmod +x)
bash hello_world.sh

3. Process Signals

  • Ctrl + C: Sends the `SIGINT` interrupt signal, immediately terminating script execution.
  • Ctrl + Z: Sends the `SIGTSTP` stop signal, suspending script execution and placing it in the background. You can resume it later using the `fg` command.

Adding Comments

Comments help explain code logic and are completely ignored by the shell interpreter during execution.

Single-line Comments

Use the hash character (`#`) to declare a comment. Everything on the line following the `#` will be ignored:

Terminal 
# This is a single-line comment. The interpreter skips this line.
echo "Hello!" # Inline comments are also supported

Multi-line Comments

You can write multi-line comments using a Here Document (HereDoc) block redirected to a dummy command name:

Terminal 
<<comment
Everything written inside this block is treated as a comment.
You can write multiple lines here.
The block ends when the matching limiter token is found.
comment

Variables & Arrays

Managing data storage using shell variables, constant readonly containers, and indexed or associative arrays.

Declaring Variables

Shell variables store strings or numbers. Use the variable name followed by the equal sign (`=`) and the value. Do not include spaces around the equal sign.

Terminal 
# Declare a variable
NAME="Paul"

# Declare a variable using command substitution (runs the command and stores output)
SERVER_NAME=$(hostname)

# Access and print the variables (prefix name with $)
echo "Hello, my name is $NAME"
echo "Running on host: ${SERVER_NAME}"
Space Restrictions

Writing NAME = "Paul" (with spaces) will result in a syntax error because the shell will attempt to execute a command named NAME with arguments = and "Paul".

Readonly Variables

To make a variable immutable (so its value cannot be changed or unset during script execution), mark it as read-only:

Terminal 
readonly PI="3.14159"
PI="3.14" # Fails! Bash will display: "PI: readonly variable"

Standard Indexed Arrays

Indexed arrays store multiple values indexed by integers starting at zero.

1. Defining and Reading Arrays

Terminal 
# Define an array (space separated items inside parentheses)
myArray=( 1 2 Hello "Hey man" )

# Access specific indices (curly braces are required)
echo "${myArray[0]}" # Outputs: 1
echo "${myArray[2]}" # Outputs: Hello

2. Array Length & Slicing

You can query the array length or extract slices of items using specialized syntax:

Terminal 
# Get total number of elements in the array (# prefix with * index)
echo "${#myArray[*]}" # Outputs: 4

# Slice: Retrieve all elements from index 1 onwards
echo "${myArray[*]:1}" # Outputs: 2 Hello Hey man

# Slice: Retrieve 2 elements starting from index 1
echo "${myArray[*]:1:2}" # Outputs: 2 Hello

3. Appending Values

Use the operator `+=` to append new values onto an existing array:

Terminal 
myArray+=( 5 6 8 )
echo "${myArray[*]}" # Outputs: 1 2 Hello Hey man 5 6 8

Associative Arrays (Key-Value)

Associative arrays store key-value pairs where keys can be strings. In Bash, associative arrays must be explicitly declared first using `declare -A`:

Terminal 
# Declare associative array explicitly
declare -A userArray

# Populate key-value elements
userArray=( [name]=Paul [age]=20 [role]=admin )

# Retrieve element by key name
echo "User name is: ${userArray[name]}" # Outputs: Paul
echo "User is ${userArray[age]} years old." # Outputs: 20

Strings & Arithmetic

Manipulating text strings, prompting for user input, and performing math calculations.

String Operations

Bash supports several built-in parameters modifications to manipulate strings efficiently without launching sub-processes:

Terminal 
myVar="Hello World!"

# 1. Retrieve string length (# prefix)
length=${#myVar}
echo "$length" # Outputs: 12

# 2. Uppercase conversion (^^ suffix)
upper=${myVar^^}
echo "$upper" # Outputs: HELLO WORLD!

# 3. Lowercase conversion (,, suffix)
lower=${myVar,,}
echo "$lower" # Outputs: hello world!

# 4. Search and Replace first occurrence (/find/replace)
replace=${myVar/World/Buddy}
echo "$replace" # Outputs: Hello Buddy!

# 5. String Slicing: ${variable:offset:length}
slice=${myVar:6:5}
echo "$slice" # Outputs: World

Taking Input from Users

Use the `read` command to halt execution and wait for user input. You can read into a variable directly or display a prompt using the `-p` flag:

Terminal 
# Simple Read (waits for typing, then stores in 'user_input')
read user_input

# Read with an inline prompt string (-p flag)
read -p "Enter your username: " NAME
echo "Greeting, $NAME!"

Arithmetic Operations

By default, Bash treats variables as strings. To calculate numeric values, use the `let` built-in command or double parentheses syntax (( ... )):

Method A: Using the `let` Command

Terminal 
a=5

# Perform calculations and assignments
let a++      # Increment a by 1 (a is now 6)
let result=a*10
echo "$result" # Outputs: 60

Method B: Using Double Parentheses

Double parentheses evaluate C-style arithmetic expressions. This is the preferred method for modern scripts because it integrates cleanly into conditional constructs:

Terminal 
a=5

# Math operations inside double parentheses
((a++))
((result=a*10))
echo "$result" # Outputs: 60

Control Flow

Directing script execution path using If statements, comparison operators, Case menus, and logical conditions.

If-Else Conditional Statements

Conditional logic relies on tests surrounded by brackets `[ ]`. Remember to leave spaces inside the brackets around your test variables.

Terminal 
marks=45

if [ $marks -gt 40 ]
then
  echo "You are PASS"
else
  echo "You are FAIL"
fi

Comparison Operators

In shell scripting, standard symbols like `>` or `<` are used for redirection. Therefore, Bash uses alphanumeric flags to evaluate integer comparisons inside bracket tests:

Operation Comparison Flag Alternative String Symbol
Equal -eq == (string check)
Not Equal -ne != (string check)
Greater Than -gt N/A
Greater Than or Equal -ge N/A
Less Than -lt N/A
Less Than or Equal -le N/A

Multiple Conditions (Elif)

Use `elif` to check sequential conditions:

Terminal 
marks=75

if [ $marks -ge 80 ]
then
  echo "First Division"
elif [ $marks -ge 60 ]
then
  echo "Second Division"
else
  echo "Fail"
fi

Case Statements (Menus)

Case statements match a single variable against multiple string options or regex patterns. Each block of statements is terminated with double semicolons `;;`, and the entire case structure ends with `esac` (case backwards):

Terminal 
echo "Choose an option:"
echo "a = See current date"
echo "b = List files in current dir"

read choice

case $choice in
  a) date;;
  b) ls;;
  *) echo "Invalid input";;
esac

Logical Operators

Evaluate compound conditions using logical operators:

  • condition1 && condition2: AND. Both conditions must be true for the compound expression to return true.
  • condition1 || condition2: OR. Returns true if at least one of the conditions is true.
  • ! condition: NOT. Reverses the boolean result of a test.

Short-circuit Chaining

You can chain commands using `&&` and `||` inline to write compact, branchless statements:

Terminal 
# Syntax: condition1 && condition2 || condition3
# Executes condition2 only if 1 is true. Otherwise, executes condition3.
[ $marks -gt 40 ] && echo "Passed" || echo "Failed"

Loops & File Iteration

Automating repetitive tasks using For, While, and Until loops, and parsing raw file/CSV streams.

For Loops

For loops iterate over a predefined list of words or a sequence range:

1. Basic List & Range Loop

Terminal 
# Iterate explicit numbers
for i in 1 2 3 4 5
do
  echo "Number is $i"
done

# Iterate names
for name in Raju Sham Baburao
do
  echo "Name: $name"
done

# Iterate ranges using brace expansion
for p in {1..5}
do
  echo "Val: $p"
done

2. Iterating Words from a File

You can load file contents and loop over each space-separated word using command substitution:

Terminal 
items="/home/paul/file.txt"

for item in $(cat $items)
do
  echo "Word: $item"
done

While Loops

While loops execute a block of commands repeatedly as long as the evaluation condition remains true:

Terminal 
count=0
num=5

while [ $count -le $num ]
do
  echo "Numbers are $count"
  let count++
done

Until Loops

Until loops are the inverse of while loops: they execute repeatedly as long as the test condition is false, halting when the condition evaluates to true:

Terminal 
a=5

until [ $a -eq 1 ]
do
  echo "Value: $a"
  a=$(expr $a - 1)
done

Infinite Loops

Declare an infinite loop by testing a command that always returns success (like `true`):

Terminal 
while true
do
  echo "System active..."
  sleep 2s
done

Reading File Streams Line-by-Line

For loop iteration splits contents by space delimiters, which breaks full-sentence lines. To read file lines containing spaces, use a `while read` loop connected to file redirection:

1. Reading a Plain Text File

Terminal 
while read myVar
do
  echo "Line Content: $myVar"
done < file_name.txt

2. Reading CSV Data Fields

You can parse comma-separated columns by declaring the Internal Field Separator (`IFS`) variable local to the `read` command:

Terminal 
while IFS="," read f1 f2 f3
do
  echo "Column 1: $f1"
  echo "Column 2: $f2"
  echo "Column 3: $f3"
done < data.csv

3. Skipping CSV Header Rows

If a CSV file contains a header row, pipe the stream through `awk` to filter out the first row before feeding the loop:

Terminal 
# awk 'NR!=1 {print}' matches all rows except row index 1
cat myfile.csv | awk 'NR!=1 {print}' | while IFS="," read f1 f2 f3
do
  echo "Data: $f1, $f2, $f3"
done

Functions & Arguments

Creating reusable command blocks, managing variables scope, and parsing script command-line arguments.

Creating Reusable Functions

Functions encapsulate a block of code, reducing redundancy. Bash supports two syntax formats for declaring functions:

Terminal 
# Syntax Format 1: Using the function keyword
function myfun {
  echo "Hi, this is Syntax 1"
}

# Syntax Format 2: standard parentheses syntax (Preferred)
otherfun() {
  echo "Hello, this is Syntax 2"
}

# Call functions by their name (DO NOT write parentheses when calling)
myfun
otherfun

Function Arguments & Scope

Functions do not accept arguments inside parentheses. Instead, parameters are passed like standard script parameters and read using positional variables (`$1`, `$2`, etc.).

Always use the `local` keyword inside functions to prevent variables from leaking into the global script scope:

Terminal 
addition() {
  # Declare local variables scope
  local num1=$1
  local num2=$2
  
  local sum
  let sum=num1+num2
  
  echo "Sum of $num1 and $num2 is $sum"
}

# Call function passing arguments (separated by space)
addition 12 13

Parsing Script Arguments

When you launch a script from your command window, you can supply parameter arguments (e.g. `./script.sh arg1 arg2`). Access these arguments using positional variables:

Special Arguments Variables

  • $#: Returns the total count of arguments passed to the script.
  • $@: Returns a list containing all arguments passed to the script.
  • $1, $2, ...: Accesses the specific argument at position index 1, 2, etc.

Looping and Shifting Arguments

Terminal 
# Print total count
echo "Total arguments: $#"

# Loop through all arguments
for arg in $@
do
  echo "Argument is: $arg"
done

# Shifting arguments: drops $1, shifts other arguments left
# If args were: A B C
shift
# Now args are: B C
echo "First argument is now: $1" # Outputs: B

Advanced Shell Programming

Deep dive into built-in signals, file tests, system variables, redirects, logging, debugging, and background operations.

Loop Flow Controls

  • break: Exits the parent loop immediately.
  • continue: Skips remaining commands in the current loop iteration and starts the next iteration.

Sleep, Exit & Exit Status ($?)

Use `sleep` to pause execution, and `exit` to terminate script execution instantly. Every command returns an exit status (from 0 to 255) indicating its result:

Terminal 
sleep 1s # Pauses script for 1 second

exit 0   # Terminates script. Status 0 indicates success.

# Query the status code of the last run command using $?
ls /nonexistent_directory
echo "Exit status: $?" # Outputs a non-zero integer (usually 2), indicating failure.

File and Directory Checks

Evaluate file path existence and type using test operators:

Terminal 
# Test if a directory exists (-d flag)
if [ -d /home/paul/scripts ]
then
  echo "Directory exists"
fi

# Test if a directory does not exist (! prefix)
if [ ! -d /home/paul/scripts ]
then
  echo "Directory does not exist"
fi

# Test if a file exists (-f flag)
if [ -f backup.tar.gz ]
then
  echo "File exists"
fi

Special System Variables

Bash pre-populates several internal variables containing runtime metadata:

  • $0 (or ${0}): The name of the executing script itself.
  • $RANDOM: Returns a random integer between 0 and 32767 on each read.
  • $UID: Returns the User ID numeric key of the current logged-in user (root is always 0).

Redirections & /dev/null

Use `>` to redirect and overwrite stdout to files, and `>>` to append. To discard output completely, redirect it to the null virtual device:

Terminal 
# Discard both standard output and error output using &> /dev/null
cd /root &> /dev/null

Logging Messages

Integrate messages into standard system syslog files using the `logger` utility. Logs are typically recorded under `/var/log/messages`:

Terminal 
logger "Sysadmin backup script initialized"

Debugging Scripts

Place control flags at the top of your script to configure execution behavior and assist in troubleshooting:

  • set -x: Command Tracing. Prints each command and its evaluated arguments to stdout before executing it.
  • set -e: Exit Immediately. Halts script execution immediately if any command returns a non-zero exit status.

Background execution (nohup)

To keep a script running in the background even if you close the terminal window or disconnect your session, prefix with `nohup` (no hangup) and append an ampersand (`&`):

Terminal 
nohup ./heavy_job.sh &

Automation & Scheduling

Scheduling one-time tasks using AT, and configuring recurring operations via Crontab cron systems.

One-Time Tasks (AT)

The `at` utility schedules command blocks to execute exactly once at a specific date and time:

1. Scheduling a Command

Type `at` followed by the execution time, enter your command list, and press Ctrl + D to save:

Terminal 
at 12:09 PM
at> ./backup.sh
at> Ctrl + D

2. Querying and Deleting Jobs

Terminal 
# List scheduled jobs
atq

# Remove a scheduled job by its numeric ID
atrm 3

Recurring Tasks (Crontab)

The system cron service executes tasks repeatedly at defined times based on configurations in the user's crontab file:

1. Crontab Commands

Terminal 
# List all active cron jobs
crontab -l

# Open crontab in text editor
crontab -e

2. Cron Syntax Fields

Each cron entry consists of 5 cron time fields followed by the command to execute:

Terminal 
# Run backup script at 12:00 AM every single day:
0 0 * * * cd /home/paul/scripts && ./create_file.sh

Practical Sysadmin Projects

Write and deploy three real-world administrative automation scripts.

Project 1: Monitoring Free RAM Space

This script parses outputs from the system memory command `free` to check available RAM and outputs a warning alert if the free memory is below a defined threshold (e.g. 500 MB):

ram_monitor.sh 
#!/bin/bash

# Define minimum memory threshold (in Megabytes)
THRESHOLD=500

# Parse available memory using free command, grep, and awk
FREE_RAM=$(free -m | grep "Mem:" | awk '{print $4}')

echo "Active system free memory: ${FREE_RAM}MB"

if [ $FREE_RAM -lt $THRESHOLD ]
then
  echo "WARNING: Active free memory is critically low!"
  logger "CRITICAL ALERT: System RAM is low. Free RAM: ${FREE_RAM}MB"
else
  echo "Memory status is within healthy boundaries."
fi

Project 2: Disk Space Monitor with Email Alerts

This script queries partitions usage statistics using `df -H`. If partition usage exceeds 80%, it compiles a detailed alert warning and sends it to the sysadmin email address:

disk_alert.sh 
#!/bin/bash

# Set threshold percentage value
THRESHOLD=80
ADMIN_EMAIL="admin@company.com"

# Query disk usage and loop through results, skipping header row
df -H | awk 'NR!=1 {print $5 " " $1}' | while read usage partition
do
  # Strip percentage sign to get numeric value
  pct_val=${usage/\%/}
  
  if [ $pct_val -ge $THRESHOLD ]
  then
    echo "WARNING: Partition $partition is at ${pct_val}% usage!"
    # Send email alert using mail command
    echo "Disk space critical on $partition (${pct_val}%)" | mail -s "Disk Space Alert: $partition" $ADMIN_EMAIL
  fi
done
